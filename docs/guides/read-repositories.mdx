# read any repository

pdsx can read public records from any repository without authentication using the `-r` flag.

## basic usage

### read with handle

```bash
pdsx -r jlowin.dev ls app.bsky.feed.post --limit 5
```

pdsx resolves:
1. handle → DID (via DNS TXT or HTTPS well-known)
2. DID → PDS url (from DID document)
3. fetches records from PDS

### read with DID (more durable)

```bash
pdsx -r did:plc:44ybard66vv44zksje25o7dz ls app.bsky.feed.post --limit 5
```

DIDs are permanent identifiers - they don't change if user updates their handle.

## when to use handles vs DIDs

**use handles for**:
- interactive exploration
- user-facing displays
- short-term references

**use DIDs for**:
- automation and scripts
- long-term storage
- reliable references

<Tip>
Handles can change or be reassigned. If you need a durable reference, use DIDs.
</Tip>

## pagination

collections can contain thousands of records. pdsx paginates results:

```bash
# first page
pdsx -r jlowin.dev ls app.bsky.feed.post --limit 10

# output includes cursor (on stderr for json/yaml):
# next page cursor: 3lyqmkpiprs2w

# next page
pdsx -r jlowin.dev ls app.bsky.feed.post --limit 10 --cursor 3lyqmkpiprs2w
```

**important**: cursors are opaque tokens, not page numbers or timestamps. they can expire after minutes.

## output formats

### json (for jq pipelines)

```bash
pdsx -r jlowin.dev ls app.bsky.feed.post -o json | jq -r '.[].text'
```

cursor goes to stderr, so json on stdout is valid for piping.

### yaml

```bash
pdsx -r jlowin.dev ls app.bsky.actor.profile -o yaml
```

### table (human-readable)

```bash
pdsx -r jlowin.dev ls app.bsky.feed.post -o table --limit 5
```

### compact (default)

```bash
pdsx -r jlowin.dev ls app.bsky.feed.post
```

shows key info without table formatting.

## practical examples

### read someone's bio

```bash
pdsx -r jlowin.dev ls app.bsky.actor.profile -o json | jq -r '.[0].description'
```

### get recent posts as json

```bash
pdsx -r did:plc:o53crari67ge7bvbv273lxln ls app.bsky.feed.post --limit 3 -o json
```

### inspect pagination cursors

for json/yaml formats, cursors appear on stderr:

```bash
pdsx -r jlowin.dev ls app.bsky.feed.post -o json 2>&1 | grep cursor
```

## troubleshooting

**handle resolution fails**

- check DNS: `dig TXT _atproto.handle`
- try HTTPS: `curl https://handle/.well-known/atproto-did`
- fall back to DID if handle resolution is broken

**cursor expired**

cursors typically expire after minutes. restart from the beginning without `--cursor`.

**empty results**

- verify collection exists: `app.bsky.feed.post` not just `post`
- check if repository is public
- try with DID instead of handle
